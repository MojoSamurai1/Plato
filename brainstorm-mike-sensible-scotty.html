Smart Flashcards + Spaced Repetition — The SM-2 algorithm is well-documented and straightforward to implement. Socratic Tutor Mode — This is primarily a prompt engineering challenge. The backend stores conversation history in a plato_conversations table. Each message has a role (user/assistant), content, and a reference to the topic being discussed. The system prompt instructs the LLM to use Socratic questioning. The file goes to WordPress via the REST API, gets stored in wp-content/uploads. Modified Features — Quick Takes:

Streaks + Progress: Simple. A plato_study_sessions table with timestamp, duration, and topic. Frontend calculates streak from consecutive days with logged sessions. Progress bars are just SQL aggregations against the mastery tracking we already have in the schema.

Wellbeing Mood Check: Add a mood_rating column (1-5) to the study sessions table. Before each session, a one-tap mood selector. If mood is low (1-2), the study plan adjusts to lighter review rather than new material. Simple conditional logic.

Round 5The Shortlist + Scotty’s Quick Specs
P1 — Foundation

📡 Canvas Sync Engine

Plato connects to Canvas LMS and automatically pulls the student’s courses, assignments, due dates, and course enrolments. This is the data backbone — without it, Plato is flying blind. The student authenticates once via a personal access token, and Plato stays in sync.

🔧 Scotty’s Spec

Student enters a Canvas personal access token in the PWA setup wizard. Token stored encrypted in WordPress usermeta. A WP-Cron job runs every 6 hours calling Canvas REST API endpoints: /api/v1/courses, /api/v1/courses/:id/assignments. Data normalised into plato_courses and plato_assignments tables. The PWA reads via plato/v1/courses and plato/v1/assignments endpoints. Fallback: manual course entry if Canvas API access is restricted by the institution.

P2 — Core Loop

🧠 The Socratic Tutor (+ ELI5 Mode)

The heart of Plato. When the student asks a question, Plato doesn’t just give the answer — it guides her to discover it herself through questioning, hints, and scaffolding. Adapts difficulty based on her mastery level. The ELI5 button re-explains any concept using simple everyday analogies.

🔧 Scotty’s Spec

Chat interface in the PWA using a streaming response pattern (SSE or chunked transfer from the WP backend). Conversations stored in plato_conversations (id, user_id, course_id, topic) and plato_messages (conversation_id, role, content, timestamp). System prompt template includes student mastery level and course context pulled from plato_study_notes. LLM calls proxied through a new class-llm.php that abstracts the provider (OpenAI/Anthropic). ELI5 mode appends an instruction to the system prompt — same endpoint, different prompt modifier. Rate-limit to 50 messages/hour to manage costs.

P3 — Content Pipeline

📚 Slide & Document Ingestion

Upload lecture slides (PDF, PPTX) and Plato extracts, summarises, and indexes the content. This feeds every other AI feature — the tutor knows what was taught, the flashcard generator has source material, and the draft reviewer understands the course content deeply.

🔧 Scotty’s Spec

File upload via the PWA (drag-and-drop zone, max 20MB). Files POST to plato/v1/documents/upload, stored in wp-content/uploads/plato/. A background processor (triggered by WP-Cron or wp_schedule_single_event) extracts text using smalot/pdfparser for PDF and PhpOffice/PhpPresentation for PPTX. Text chunked into ~1000-token segments, each sent to LLM for summarisation. Results stored in plato_study_notes (id, course_id, document_id, chunk_index, raw_text, summary, key_concepts_json). Key concepts extracted as structured JSON for flashcard generation.

P4 — Retention

⚡ Smart Flashcards with Spaced Repetition

Auto-generated flashcards from course content, served using the SM-2 spaced repetition algorithm. Cards appear at the optimal moment before the student would forget the concept. No manual card creation needed — Plato builds the deck from ingested materials.

🔧 Scotty’s Spec

When documents are ingested (P3), a secondary LLM pass generates Q&A pairs stored in plato_flashcards (id, course_id, document_id, question, answer, card_type). Review state in plato_card_reviews (card_id, user_id, ease_factor default 2.5, interval, repetitions, next_review_date). SM-2 algorithm implemented in class-spaced-repetition.php. PWA endpoint plato/v1/flashcards/due returns cards where next_review_date <= NOW() ordered by priority. After review, PWA sends quality rating (0-5), backend recalculates schedule. Daily review session takes ~10-15 minutes.

P5 — Assignments

📝 Draft Reviewer

Paste in an assignment draft and Plato reviews it against the rubric. Identifies gaps, weak arguments, missing rubric criteria, and structural issues. Never writes content — only gives actionable feedback. Like having a tutor available at 11pm on a Sunday night.

🔧 Scotty’s Spec

New PWA page at /review with a text input (max 10,000 chars) and optional rubric input (pulled from Canvas if available, or manually entered). POST to plato/v1/review/draft. Backend constructs a prompt with rubric criteria and strict instructions to review-not-write. Response stored in plato_draft_reviews (id, user_id, assignment_id, draft_text_hash, feedback, rubric_criteria_json, created_at). Feedback is structured: overall assessment, per-criterion feedback, and top-3 improvement suggestions. Academic integrity guardrail enforced at the prompt level.

P6 — Engagement

🔥 Study Streaks & Session Tracking

Simple but powerful motivation mechanics. Track study sessions, maintain daily streaks, log time per topic. Feeds the learning dashboard with real data about study habits. Includes a one-tap mood check before each session so Plato can adjust the study plan when she’s having a rough day.

🔧 Scotty’s Spec

Study timer component in the PWA (React state, no external deps). On session end, POST to plato/v1/sessions with { course_id, topic, duration_minutes, mood_rating (1-5), session_type }. Stored in plato_study_sessions. Streak calculated server-side: consecutive calendar days with at least one session. Exposed via plato/v1/stats/streak. Learning dashboard aggregates: total hours this week, hours per course, mastery progression, mood trend. If mood_rating <= 2, the next flashcard session serves only review cards (no new material).

P7 — Nudges

🔔 Smart Notifications

Three types only: assignment due reminders (3 days and 1 day before), retention decay alerts (“You haven’t reviewed Marketing in 9 days”), and a weekly study summary. Context-aware, respectful of attention, and genuinely helpful.

🔧 Scotty’s Spec

PWA registers a service worker for Web Push API. User grants notification permission during onboarding. Push subscription stored in plato_push_subscriptions (user_id, endpoint, p256dh, auth). WP-Cron job runs daily at 8am checking: (1) assignments due within 3 days or 1 day, (2) flashcard decks with no review in 7+ days, (3) on Sundays, compile weekly summary. Notifications sent via minishlink/web-push PHP library. Payload is a small JSON object rendered by the service worker. User can configure quiet hours in PWA settings.
